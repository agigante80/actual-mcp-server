## Per-branch detail (diverged branches with novel content)

### dsqr84 : develop  (analysed 2026-07-10, head `432ac97`)

- What diverged: six OIDC commits by @dsqr84. Five were already harvested and shipped:
  JWKS discovery / clean-401 / crash-on-expired-JWT / sessionPrincipals cleanup (#244,
  v0.7.10) and the audience allowlist (#245). The branch grew by two newer commits
  that add the RFC 8414 angle.
- Novel functionality upstream lacks (ticketed as #285):
  - **RFC 8414 Authorization Server Metadata endpoint** (`src/server/httpServer.ts`,
    commits `5d0a659` and `432ac97`): serves `/.well-known/oauth-authorization-server`
    on the resource-server origin so `mcp-remote` and Claude.ai's native OAuth connector
    can discover the `token_endpoint`, synthesized from the issuer's OIDC discovery
    document. Needed because some IdPs (Authentik) do not expose that well-known path.
    We already serve the complementary RFC 8707/9728 protected-resource metadata
    (`/.well-known/oauth-protected-resource`) but not this. Idea value: HIGH (unlocks the
    built-in Claude.ai OAuth flow against standard OIDC IdPs). Security-sensitive (SSRF,
    endpoint disclosure): mandatory security review before adoption.
- Noise (ignored): `npm audit` dependency bump, `docker-compose.yaml`/`package.json` tweaks.
- `dsqr84:main` (head `f7cfcb3`, 3 ahead) is an older subset of the #244-shipped fixes.

### zdev-code : main  (analysed 2026-07-10, head `c9a10af`)

- What diverged: `Add GitHub Actions workflow for GHCR build and push` + a version bump to 0.7.12.
- Verdict: no novel functionality. The only added file is `.github/workflows/ghcr-only.yml`,
  a standalone GHCR publish workflow. Our CI/CD pipeline already builds and publishes a
  multi-arch image to both GHCR and Docker Hub, so this adds nothing. No ticket.

### kstimp : feat/oidc-jwks-trusted-hosts  (analysed 2026-07-10, head `fac9bdb`)

- What diverged: a single commit `feat(auth): OIDC cross-origin JWKS support via
  OIDC_JWKS_TRUSTED_HOSTS`, touching `src/config.ts`, `src/lib/oidc-discovery.ts`,
  `src/server/httpServer.ts`, docs, and the JWKS discovery test.
- Verdict: SHIPPED upstream. This is exactly our #254 (`OIDC_JWKS_TRUSTED_HOSTS`, an
  opt-in cross-origin JWKS host allowlist), already released and present in
  `src/lib/oidc-discovery.ts` / `src/config.ts`. Same feature, no open idea. No ticket.

## How this file is maintained

Run the `fork-analysis` skill (ask to "analyse the forks" or "scan forks for
features"). It enumerates every fork and each meaningful branch (excluding
`dependabot/*`), re-checks divergence, re-analyses only branches whose head sha
changed since the rows above, carries forward unchanged verdicts, and rewrites
this file. It never clones or runs fork code; it reads the GitHub API and file
contents only.
